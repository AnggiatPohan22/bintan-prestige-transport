# Frontend Guidelines

## Direction

Build a modern, clean, premium taxi and travel frontend that feels trustworthy,
fast, and easy to use from mobile devices.

## UI Rules

- Use mobile-first layout decisions.
- Keep cards reusable and visually restrained.
- Use dark charcoal, white, slate neutrals, and soft gold accents.
- Keep border radius at 8px unless a button or pill needs a rounded touch target.
- Use strong image areas for travel context when real assets are added.
- Do not build marketing-only sections before the actual page structure exists.

## Component Rules

- Shared primitives belong in `src/components/common`.
- Homepage-specific sections belong in `src/components/home`.
- Package-specific components belong in `src/components/packages`.
- React components should be reserved for interactivity.
